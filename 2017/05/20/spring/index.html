spring

发表于 2017-05-20   |   分类于 SSH   |     |   阅读次数

概述

定义

Spring是一个JavaSE/EE的一栈式(full-stack)轻量级开源框架.
一栈式是指:Spring框架在JavaEE每层都有解决方案
Web: SpringMVC;
Service: Spring管理Bean,Spring管理事务;
Dao:Spring的JDBC模版,Spring的ORM模块;

作用

Spring的核心功能就是对象的”容器”,用来管理项目中的所有对象(横跨三层).
说Spring能够处理请求,是因为Spring管理了可以处理请求的对象
说Spring能够操作数据库,是因为Spring管理了可以操作数据库的对象.
特点:

  • 方便解耦,简化开发;
  • 支持AOP编程;
  • 支持声明式事务管理;
  • 支持Junit测试;
  • 方便集成各种优秀框架;
  • 简化API应用难度;

    相关概念

    IOC

    Inverse Of Control反转控制,是指在Spring中,对象的创建方式被反转了:
    不使用Spring时,对象由开发人员手动创建并获得;
    使用Spring时,对象由Spring(程序)帮我们创建并获得

    DI

    Dependency Injection依赖注入,是指将属性值注入对象的技术.
    不使用Spring时,依赖注入由开发人员手动完成;
    使用Spring时,依赖注入由Spring(程序)帮我们完成.
    Spring支持的注入技术:
    1.构造方法注入 –> 调用构造方法完成注入
    2.set方法注入 –> 调用set方法完成注入
    注意: 实现反转控制需要依赖注入技术的支持.

    BeanFactory&ApplicationContext

    BeanFactory(旧),在需要获得对象时才创建对象,现在一般不用
    ApplicationContext(新),在容器创建时,默认创建所有对象,获得时直接取出.
