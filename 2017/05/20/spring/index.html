发表于 2017-05-20   |   分类于 SSH   |     |   阅读次数

概述

定义

Spring是一个JavaSE/EE的一栈式(full-stack)轻量级开源框架.
一栈式是指:Spring框架在JavaEE每层都有解决方案
Web: SpringMVC;
Service: Spring管理Bean,Spring管理事务;
Dao:Spring的JDBC模版,Spring的ORM模块;

作用

Spring的核心功能就是对象的”容器”,用来管理项目中的所有对象(横跨三层).
说Spring能够处理请求,是因为Spring管理了可以处理请求的对象
说Spring能够操作数据库,是因为Spring管理了可以操作数据库的对象.
特点:
方便解耦,简化开发;
支持AOP编程;
支持声明式事务管理;
支持Junit测试;
方便集成各种优秀框架;
简化API应用难度;

相关概念

IOC

Inverse Of Control反转控制,是指在Spring中,对象的创建方式被反转了:
不使用Spring时,对象由开发人员手动创建并获得;
使用Spring时,对象由Spring(程序)帮我们创建并获得

DI

Dependency Injection依赖注入,是指将属性值注入对象的技术.
不使用Spring时,依赖注入由开发人员手动完成;
使用Spring时,依赖注入由Spring(程序)帮我们完成.
Spring支持的注入技术:
1.构造方法注入 –> 调用构造方法完成注入
2.set方法注入 –> 调用set方法完成注入
注意: 实现反转控制需要依赖注入技术的支持.

BeanFactory&ApplicationContext

BeanFactory(旧),在需要获得对象时才创建对象,现在一般不用
ApplicationContext(新),在容器创建时,默认创建所有对象,获得时直接取出.

Spring中的aop开发理论

AOP

面向切面编程.AOP出现是OOP(Object Oriented Programming,OOP,面向对象程序设计)的延伸,是OOP的扩展,解决OOP中的一些问题.
AOP的底层实现原理是代理机制,Spring希望可以对所有类使用代理技术进行代理,Spring实现AOP使用两种代理机制：JDK动态代理和CGlib代理

代码准备

利用UserService和UserServiceImpl来演示,被代理类是UserServiceImpl,因为实现了UserService接口,所以动态代理和CGLib都可以进行代理.

JDK动态代理

特点: 只能对实现接口的类进行代理,如果没有实现接口,无法进行代理

代理工厂类: 用于返回代理对象.可以选择实现InvocationHandler接口,或者在newProxyInstance方法的第三个参数利用匿名内部类实现该接口,重写invoke方法

CGlib代理类

CGlib既可以代理实现接口的类,也可以代理未实现接口的类.代理类和被代理类是继承关系

AOP中的相关术语

连接点(join point)

目标对象中,所有可以增强的方法.比如上例UserServiceImpl的增删改查方法都可以进行增强

切入点(point cut)

即将要增强或已经增强的方法,比如只增强save

目标对象(target)

被代理对象.比如上例的UserServiceImpl

通知(advice)

要对目标对象的切点增强的代码,比如增加的事务管理的代码

代理对象(proxy)

对目标对象的切点应用通知后生成的对象

织入(weaving)

将通知应用到目标对象切点的过程

切面(aspect|advisor)

通知和切点合在一起,称为切面
