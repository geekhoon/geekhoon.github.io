1.HashMap和Hashtable的区别:

* 共同点: 
    1. 底层都是哈希算法
    2. 都是双列集合
* 区别一: 
    1. HashMap   JDk1.2出现, 线程不安全集合, 效率较高
    2. Hashtable JDk1.0出现, 线程安全集合,   效率较低 
* 区别二: 
    1. HashMap 允许存储null键,null值
    2. Hashtable 不允许存储null键,null值

4.什么是面向对象?
面向对象是一种编程思想,是基于面向过程的
面向过程是强调过程,每一步都要亲自完成
面向对象是强调对象,让对象去帮助完成
好处:

    * 更符合人类思考习惯
    * 将复杂的问题简单化
    * 由执行者变成指挥者
三大特性:
1. 封装:
    隐藏对象的属性和实现细节,对外提供公共的访问方式
    好处: 
        * 提高了代码复用性
        * 提高了安全性
        * 便于调用者使用
2. 继承:
    当多个类中有很多共性内容时,可以把这些共性内容抽取出来封装成一个类,让这些类与这个封装的类产生关系,就是继承
    特点:
        * 类与类之间的继承: 只能单继承,不能多继承,但是可以多层继承
        * 接口与接口之间的继承: 可以单继承,也可以多继承
    好处:
        * 提高了代码的复用性
        * 提高了代码的维护性
        * 提高了代码扩展性
        * 类与类之间产生了关系, 是多态的前提
    坏处:
        * 增强了类与类之间的耦合性
3. 多态:
    同种事物在不同时刻表现出来的不同形态
    前提:
        * 要有继承(实现)关系
        * 要有方法重写
        * 父类或父接口引用指向子类对象
    好处:
        * 提高了代码的维护性
        * 提高了代码的扩展性
    坏处:
        * 父类引用不能直接调用子类特有的属性和方法
    成员访问特点:
        * 普通方法
        * 成员变量
        * 静态方法

5.局部变量和成员变量的区别
1.定义位置不同
局部变量：方法中或方法声明上
成员变量：类中方法外
2.作用域不同
局部变量：方法中，出了方法就不能用了
成员变量：整个类
3.默认值不同
局部变量：没有默认值，局部变量必须先赋值才能使用
成员变量：有默认值。和数组元素默认值一样
4.内存位置不同
局部变量：在栈内存的方法中
成员变量：在堆内存的对象中
5.生命周期不同
局部变量：随着方法弹栈而消亡
成员变量：随着对象的回收而消亡
6.接口与抽象类的区别

1. 成员: 接口中全是抽象方法,抽象类中可以有抽象方法,也可以有非抽象方法
         接口中成员变量只能是常量,抽象类中可以有变量
2. 继承关系: 接口可以多实现,多继承   抽象类只能单继承,多层继承
3. 设计理念:  接口是这个事物中的额外内容   抽象类是这个事物中应该具备的内容 
*接口: 
    常量的默认修饰符是public static final ;
    方法的默认修饰符是：public abstract;

7.instanceof关键字: 比较引用数据类型
格式:　引用变量 instanceof 类名
8.this:

* 代表本类对象的引用
* 区分成员变量和局部变量的同名情况
* 在本类中调用其他构造方法

9.final:

1. 修饰类: 不可以被继承,但是可以继承别的类
2. 修饰方法: 不可以被子类重写,但是可以被继承使用
3. 修饰局部变量:
    1. 修饰基本数据类型: 称为常量,只能被赋值一次
    2. 修饰引用数据类型: 变量存储的地址不可改变,但指向的对象属性值可以改变
4. 修饰成员变量:
    1. 需要在创建对象前赋值,不能用set赋值
    2. 固定的是成员变量的手动赋值,不是默认值

10.static:

1. 修饰方法: 不能被重写,可以重载
2. 修饰成员变量
* 特点: 
    * 静态成员随着类的加载而加载
    * 优于对象存在
    * 能够通过类名点的形式直接调用
    * 修饰的成员属于类, 被该类的所有对象共享
* 注意:
    * 静态方法中不可以定义this、super关键字
    * 静态不能直接访问非静态
* 好处:
    * 能够被类名点的形式调用，简化书写。被该类下所有的对象所共享
* 坏处:
    * 生命周期过长

11.修饰类只能使用public、default、final、abstract关键字
abstract不能和private,static,final一起用
12.代码块:

1. 局部代码块: 限定局部变量的生命周期'
2. 构造代码块: 在创建对象的时候执行，每创建一次，就执行一次。构造代码块优先于构造方法执行
3. 静态代码块: 在类加载的时候执行，并且只会执行一次，优先于构造代码块执行
4. 同步代码块

13.抽象类和抽象方法的特点:
抽象类不能实例化，需要子类继承；抽象方法没有方法体，需要子类去重写
14.异常处理有哪些方式?
（1）try-catch
（2）throws
15.运行异常和编译异常的区别？
运行时异常不处理也能通过编译，RuntimeException及其子类都是运行时异常。
而编译期异常必须处理，才能通过编译。非RuntimeException及其子类都是编译期异常。
