MySQL

MySQL_排序

select * from 表名 [where 条件] [order by 字段名 asc(升序)/desc(降序)]

MySQL_聚合函数

count(字段名/*); 统计指定列不为null的行数(任意类型);

sum(字段名); 计算指定列的数值和(数值);

MAX(字段名); 求指定列的最大值(数值,日期);

MIN(字段名); 求指定列的最小值(数值,日期);

AVG(字段名); 求指定列的平均值(数值);

注意:

对于使用了”聚合函数”查询,其结果列应该只有聚合函数的一个结果列,不应该再包含其它列.不保证其它列跟”聚合结果列”是同一条记录;

MySQL_分组

select * from 表名 [where 条件] [group by 字段名] [having 条件]; 将列”值”相同的作为一组,然后在分组的基础上可以进行聚合查询;

注意:

  1. 分组查询的结果中只能包含分组列和聚合列,不能包含其它列;
  2. 分组查询之后只能使用having 条件 对结果进行筛选;

MySQL_分页

select from 表名 [where 条件] limit m, n; m: 开始索引,n: 每页大小(m = (当前页 - 1) n);

注意

m与n不能为负数;

MySQL_执行顺序

  1. from
  2. where
  3. group by
  4. having
  5. select
  6. order by
  7. limit

MySQL_多表

设计表原则:

每张表只记录一件事情,若一张表记录了多件事情,就要拆分成多表存储,然后利用外键来引用另一张表中的记录;
主表: 被其它表引用的表;
从表: 引用其它表的表;

表与表之间的关系:

  1. 一对多(最常见)

    客户 – 订单
    一位客户可以有多张订单;
    一张订单只属于一位客户;
    分类 – 商品
    一类分类可以有多种商品;
    一种商品只属于一类分类;

  1. 多对多(较常见)

    学生 – 课程
    一位学生可以学多门课程
    一门课程可以被多位学生学习

  2. 一对一(尽量不见)

外键约束:

外键约束的作用:

对”从表”的外键字段添加”外键约束”,可以让数据库软件自动检查

声明外键约束

alter table 从表 add [constraint] [外键名称] foreign key (从表外键字段名) references 主表 (主表主键字段名);

删除外键约束

alter talbe 从表 drop foreign key 外键名称;

多表查询语法

交叉查询(笛卡尔集)

select * from 表A, 表B; 查询出的结果是错误的;

内连接查询

隐式内连接查询

select * from 从表, 主表 where 从表.外键字段 = 主表.主键字段;

显式内连接查询

select * from 从表 inner join 主表 on 从表.外键字段 = 主表.主键字段;

外连接查询

左外连接查询

select * from 左表 left outer join 右表 on 等值条件;
查询左表所有的数据,右表是否显示,取决于条件是否成立,如果成立显示,如果不成立显示null;

右外连接查询

select * from 左表 right outer join 右表 on 等值条件;
查询右表所有的数据,左表是否显示,取决于条件是否成立,如果成立显示,如果不成立显示null;

子查询

一条select查询的结果可以作为另一条select查询的查询条件或查询表等信息;
